# TASK-02 ヒント: ステータスフィルタ

## 実装の入口
- API の `GET /api/products` に `status` クエリ (`active` / `inactive`) を追加。
- 一覧画面に `<select>` を置いて選択値を API に渡します。

## API側のヒント
- `status` が不正値の場合は無視して全件返却にするか、422にするか方針を先に決めます。
- 教材では「不正値は無視」が扱いやすいです。

## フロント側のヒント
- URL クエリに `status` を反映させると再読み込みに強くなります。
- 検索と同時導入するなら、API引数を object にしておくと拡張しやすいです。
